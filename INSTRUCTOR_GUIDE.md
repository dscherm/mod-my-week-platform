# CyberEd Range - Instructor Guide

## Overview

CyberEd Range is a comprehensive cybersecurity education platform designed for high school students. This guide will help instructors integrate the platform into their curriculum and maximize student learning outcomes.

## Educational Alignment

### Learning Standards

The platform aligns with:
- **NICE Framework**: Cybersecurity education competencies
- **CSTA Standards**: K-12 Computer Science Standards
- **Common Core**: Critical thinking and problem-solving
- **CTE Standards**: Career and Technical Education in IT Security

### Grade Level

- **Recommended**: Grades 9-12
- **Prerequisites**: Basic computer literacy
- **Course Type**: Cybersecurity, Computer Science, or IT elective

## Challenge Categories & Learning Objectives

### 1. Cryptography (5 Challenges, 80 Points)

**Learning Objectives:**
- Understand encryption vs encoding
- Learn classical ciphers and cryptanalysis
- Recognize encoding schemes (Base64, Hex)
- Understand cryptographic hashing
- Differentiate symmetric and asymmetric encryption

**Key Vocabulary:**
- Encryption, Decryption, Cipher, Plaintext, Ciphertext
- Caesar Cipher, Base64, Hash

**Challenges:**
1. Caesar Cipher Basics (Easy, 10pts)
2. Base64 Encoding (Easy, 15pts)
3. Hex Decoder (Easy, 15pts)
4. ROT13 Message (Medium, 20pts)
5. Hash Detective (Medium, 25pts)

### 2. Network Security (4 Challenges, 85 Points)

**Learning Objectives:**
- Identify common network ports and services
- Understand firewall protection
- Learn network protocols (HTTP, HTTPS, SSH, FTP)
- Recognize suspicious traffic patterns
- Understand IP addressing

**Key Vocabulary:**
- Firewall, Port, IP Address, Packet, Protocol

**Challenges:**
1. Common Ports (Easy, 10pts)
2. IP Address Classes (Medium, 20pts)
3. Protocol Detective (Medium, 25pts)
4. Firewall Rules (Hard, 30pts)

### 3. Password Security (4 Challenges, 80 Points)

**Learning Objectives:**
- Learn password strength criteria
- Understand brute force attacks
- Recognize password patterns and weaknesses
- Learn about password managers and 2FA
- Understand password hashing and salting

**Key Vocabulary:**
- Brute Force Attack, Password Strength, Authentication
- Hash, Encryption

**Challenges:**
1. Password Strength 101 (Easy, 10pts)
2. Brute Force Math (Medium, 20pts)
3. Common Password Patterns (Medium, 20pts)
4. Password Hashing (Hard, 30pts)

### 4. Web Security (4 Challenges, 75 Points)

**Learning Objectives:**
- Identify web vulnerabilities (XSS, SQL injection)
- Understand input sanitization
- Learn secure coding practices
- Recognize malicious URLs and phishing
- Understand HTTPS and SSL/TLS

**Key Vocabulary:**
- SQL Injection, XSS (Cross-Site Scripting)
- Vulnerability, Exploit, Protocol, Encryption

**Challenges:**
1. URL Analysis (Easy, 10pts)
2. HTTPS vs HTTP (Easy, 15pts)
3. SQL Injection Basics (Medium, 25pts)
4. XSS Detection (Medium, 25pts)

### 5. Social Engineering (4 Challenges, 75 Points)

**Learning Objectives:**
- Recognize social engineering tactics
- Understand phishing indicators
- Learn about pretexting and manipulation
- Develop critical thinking for security awareness
- Understand the human element in cybersecurity

**Key Vocabulary:**
- Phishing, Social Engineering
- Authentication, Authorization, Malware, Exploit

**Challenges:**
1. Phishing Email Detection (Easy, 10pts)
2. Pretexting Scenario (Medium, 20pts)
3. Tailgating Attack (Medium, 20pts)
4. USB Baiting (Hard, 25pts)

## Curriculum Integration

### Suggested Course Structures

#### Option 1: Full Semester Course (18 weeks)

- **Weeks 1-2**: Introduction to Cybersecurity
- **Weeks 3-5**: Cryptography unit + challenges
- **Weeks 6-8**: Network Security unit + challenges
- **Weeks 9-11**: Password Security unit + challenges
- **Weeks 12-14**: Web Security unit + challenges
- **Weeks 15-17**: Social Engineering unit + challenges
- **Week 18**: Final CTF competition

#### Option 2: Mini-Unit (4-6 weeks)

- **Week 1**: Intro + Vocabulary + Easy challenges
- **Week 2-3**: Medium difficulty challenges
- **Week 4-5**: Hard challenges + projects
- **Week 6**: Assessment and presentations

#### Option 3: After-School Club or Camp

- **Session-based**: One category per meeting
- **Self-paced**: Students progress independently
- **Competitive**: Weekly leaderboards

### Assessment Strategies

#### Formative Assessment
- Monitor challenge completion rates
- Review hint usage patterns
- Observe vocabulary retention
- Check progress on learning objectives

#### Summative Assessment
- Minimum challenge completion requirements
- Written explanations of solutions
- Vocabulary quizzes
- Practical demonstrations
- Portfolio of completed challenges

### Grading Rubric Suggestions

**Challenge Completion (40%)**
- All Easy challenges: 10%
- All Medium challenges: 15%
- All Hard challenges: 15%

**Vocabulary Mastery (20%)**
- Quizzes and definitions
- Proper usage in explanations

**Written Explanations (20%)**
- Solution documentation
- Concept understanding
- Critical thinking

**Final Project/CTF (20%)**
- Apply learned concepts
- Problem-solving skills
- Creativity

## Classroom Management

### Before Starting

1. **Preview all challenges** - Complete them yourself first
2. **Set expectations** - Discuss ethical hacking and responsible use
3. **Check equipment** - Ensure all students have computer access
4. **Create accounts** - Each student uses their own browser/profile

### During Implementation

1. **Progress monitoring** - Check student advancement regularly
2. **Hint management** - Encourage problem-solving before hints
3. **Collaborative learning** - Allow peer discussion, not answer sharing
4. **Differentiation** - Adjust difficulty expectations by student level

### Best Practices

- **Start with easier challenges** - Build confidence
- **Encourage exploration** - Learning through trial and error
- **Discuss vocabulary** - Reinforce terms regularly
- **Real-world connections** - Relate to current events and news
- **Ethics emphasis** - Always discuss legal and ethical implications

## Troubleshooting

### Common Issues

**Students stuck on challenges:**
- Encourage hint usage
- Facilitate peer discussion
- Provide additional scaffolding
- Allow collaborative problem-solving

**Varying skill levels:**
- Differentiate completion requirements
- Offer extension challenges
- Pair programming/mentoring
- Self-paced progression

**Ethical concerns:**
- Emphasize legal boundaries
- Discuss responsible disclosure
- Explain white hat vs black hat
- Require acceptable use agreement

## Support and Feedback

### For Technical Issues
- Check browser console for errors
- Verify localStorage is enabled
- Try different browser
- Clear cache and restart

### For Educational Questions
- Refer to challenge explanations
- Consult external resources
- Connect with cybersecurity professionals
- Join educator communities

## Conclusion

CyberEd Range provides a solid foundation for high school cybersecurity education. By completing all challenges, students will have exposure to fundamental concepts, vocabulary, and practical skills that prepare them for further study or careers in cybersecurity.

Remember: The goal is education and awareness, not creating hackers. Always emphasize ethical use and legal boundaries!
